# 《野排模拟器》游戏设计总览

> 文档版本：0.1  
> 类型：单人、回合制、事件驱动、轻度撤离与小队管理  
> 引擎：Godot 4.x

## 1. 产品愿景

《野排模拟器》希望还原一种熟悉又荒诞的组队体验：你知道正确战术是什么，但真正困难的是让三个临时队友愿意一起做。

玩家不直接操纵所有成员，而是通过指令、信息、关系和职业能力影响队伍。AI 队友有能力，也有私心；会立功，也会抢物资、畏战、冒进、误判和互相甩锅。玩家的乐趣来自阅读这些人、承担风险并把一场即将失控的行动勉强带回家。

## 2. 目标体验

### 战术判断

玩家要根据不完整信息选择进攻、搜索、稳守或撤离。

### 人性管理

同一条命令对不同队友产生不同结果。管理信任与压力和判断敌情同样重要。

### 撤离赌局

已经获得的收益只有成功撤离才真正属于玩家。继续深入永远更诱人，也更危险。

### 可复述的事故

一局结束后，玩家应该能讲出具体故事：“医生冒险救回了贪物资的队友，结果突击手以为对面残血自己冲了。”

## 3. 核心支柱

1. 有原因的不服从：AI 行为需要可解释，不能伪装成性格的纯随机。
2. 有代价的领导力：强行命令可以解决眼前问题，但会消耗关系或资源。
3. 有诱惑的撤离：收益必须持续诱惑玩家多留一个节点。
4. 横向的角色差异：职业改变解决问题的方法，不简单替代低级职业。
5. 配置驱动内容：角色、AI 模板、卡牌和事件由数据定义。

## 4. 完整游戏循环

### 局外

```text
查看行动目标
→ 选择玩家角色
→ 配置指令组与角色专长
→ 进入匹配
→ 完成对局
→ 结算带出物资、经验与角色熟练度
→ 解锁新的横向选项
```

### 局内

```text
认识随机队友
→ 观察节点信息
→ 选择指令
→ 队友作出响应
→ 结算事件
→ 管理伤势、压力、信任和物资
→ 继续深入或撤离
```

## 5. 玩家决策层

每个节点依次包含四个阶段：

1. 情报阶段：展示敌情、收益、撤离状态和队友倾向；部分信息可能模糊或错误。
2. 指令阶段：玩家选择一张指令，或使用角色能力。
3. 响应阶段：预览队友的大致意向，允许有限次数的劝说或强制。
4. 结算阶段：执行行为，更新状态并生成反馈。

玩家不应该在一次结算前完成大量微操作。单个节点目标操作时长为 20—45 秒。

## 6. 风险与收益

### 威胁来源

- 进入更深节点。
- 战斗失败或拖延。
- 搜索高价值区域。
- 队伍分散、成员倒地或制造噪音。
- 特殊事件和角色副作用。

### 收益来源

- 节点物资。
- 完成行动目标。
- 全队撤离奖励。
- 处理高风险事故。
- 角色或指令相关挑战。

### 风险曲线

前 2 个节点用于了解队友；中段让玩家积累收益并建立矛盾；后段通过撤离窗口和事故概率迫使玩家表态。最终撤离不应成为地图末端的固定按钮，而应是一个需要条件、时间或队伍协作的节点。

## 7. AI 队友设计

AI 队友由四层组成：

| 层 | 内容 | 是否跨局 |
| --- | --- | --- |
| 能力 | 战斗、支援、侦察等 | 仅本局生成 |
| 驱动力 | 物资欲、生存欲、表现欲 | 仅本局生成 |
| 人格模板 | 驱动力组合和特殊反应 | 模板永久存在 |
| 局内关系 | 信任、压力、伤势、个人物资 | 不跨局 |

同一人格不应总做同一件事。人格决定偏好，局势决定选择，玩家行为决定关系。

AI 决策反馈需要分三级展示：

- 明确信息：“阿强想搜索，因为物资欲很高。”
- 概率信息：“阿强大概率会跟随。”
- 事后解释：“阿强违抗：高价值物资 + 压力过高。”

## 8. 指令系统

指令是游戏的主要操作语言，分为：

- 战术：集火、稳守、侧袭、分散。
- 行动：搜索、救援、撤离、跟随。
- 关系：劝说、承诺分配、强制服从。
- 情报：侦察、确认目标、识别风险。

完整版可允许玩家组建 10—12 张指令组，但每局手牌或冷却机制必须保证选择清晰。角色可以替换或强化部分指令，不能完全绕过 AI 决策。

## 9. 角色系统

玩家角色代表领导风格和团队职责。

| 角色 | 强项 | 代价 | 主要改变 |
| --- | --- | --- | --- |
| 医生 | 救援、恢复、减压 | 正面战斗较弱 | 队伍容错 |
| 突击手 | 进攻、压制、建立士气 | 失败时风险更高 | 战斗节奏 |
| 侦察员 | 情报、预判、规避伏击 | 收益能力一般 | 信息质量 |
| 指挥官 | 服从、协同、稳定 | 个人能力较弱 | AI 可控性 |
| 后勤员 | 物资、分配、撤离收益 | 战斗贡献较低 | 经济与贪婪 |

角色详细数值见[角色数值与成长](CHARACTER_AND_PROGRESSION.md)。

## 10. 成长原则

### 局内成长

局内变化应制造新的选择：

- 信任提高后可使用一次高风险协同。
- 压力升高后角色能力变强，但更可能失控。
- 获得临时物资后提升战力，同时增加争夺。
- 完成事件后获得只在本局有效的专长。

### 局外成长

局外成长以横向解锁为主：

- 新角色。
- 新指令。
- 同一角色的不同专长。
- 新 AI 人格、事件和难度修正。
- 外观、档案和挑战记录。

禁止用长期无限叠加的基础伤害解决难度问题。数值纵向成长应有低上限，且主要用于早期引导。

## 11. 事件设计模板

每个事件至少包含：

```text
事件 ID
事件类型
前置条件
公开信息
隐藏信息
可用指令标签
AI 候选行为
结算规则
状态变化
反馈文本
后续事件标签
```

示例：

```text
事件：仓库里的高级物资
公开信息：高收益、中等敌情、会增加威胁
隐藏信息：存在伏击
人格影响：物资欲提高搜索效用，生存欲提高警戒效用
可能结果：安全获得 / 遭伏击 / 队友私藏 / 主动放弃
```

## 12. 难度设计

难度不只提高敌人生命，主要调整：

- 情报模糊程度。
- AI 压力增长速度。
- 撤离窗口苛刻程度。
- 高价值物资诱惑。
- 事故恢复成本。
- 敌方行为复杂度。

基础难度要允许玩家看懂系统；高难度才要求玩家利用人格、关系和构筑。

## 13. 反馈与 UI 原则

- 决策前告诉玩家“可能发生什么”。
- 决策后告诉玩家“为什么发生”。
- 重要状态同时使用数字、颜色和短文本，不只显示进度条。
- 隐藏精确概率可以保留戏剧性，但不能隐藏影响因素。
- AI 台词服务于状态反馈，不只是装饰。

建议主界面布局：

```text
顶部：节点、威胁、撤离状态、总物资
左侧：三名 AI 的生命/压力/信任/当前意向
中央：事件描述与结果演出
底部：玩家指令
右侧：玩家角色能力与本局状态
```

## 14. 版本路线

### 原型版

验证 AI 偏离、指令和撤离循环。范围以 [MVP](MVP.md) 为准。

### Alpha

加入 4—5 个角色、指令构筑、更多人格、事件链和基础存档。

### Beta

加入完整局外解锁、难度层级、内容平衡、教程和表现优化。

### 正式版候选方向

每日行动、挑战规则、角色专长组合、可分享的对局种子。联网合作或 PVP 不作为默认路线，除非单人核心已经稳定。

## 15. 待验证风险

- AI 违抗是否让玩家觉得不公平。
- 解释信息过多是否拖慢节奏。
- 职业能力是否会压过队友人格。
- 撤离收益是否足以形成“再搜一次”的诱惑。
- 随机事件是否真正改变决策，而不只是换皮。
