# 角色数值与成长系统

> 文档版本：0.3
> 目标：建立一个“玩家可以指挥与统筹，但无法稳定控制全局”的角色、AI 与成长体系。
> 设计关键词：随机队友、有限统筹、高失败率、可解释偏离、局内成长、横向局外成长。

## 1. 系统定位

《野排模拟器》的角色系统不追求传统 RPG 式的“数值越高越稳”，而是追求一种更接近野排体验的状态：

- 玩家是队伍的临时核心，但不是所有人的上级。
- 指挥可以提高成功概率，但不能消除失控。
- 队友有能力、有私心、有情绪，也会在压力下做出合理但麻烦的选择。
- 随机性必须可解释：失败可以突然发生，但事后应能说清“为什么炸了”。

角色系统分为三层：

1. 玩家干员：决定玩家擅长怎样统筹队伍。
2. AI 队友：每局随机生成，决定队伍的不稳定来源。
3. 成长系统：局内制造短期变化，局外解锁新的横向玩法。

玩家干员越强，不应代表“更容易无脑赢”，而应代表“有更多方法把快炸的局面救回来”。

## 2. 数值尺度

所有主要属性使用 `0—100`，便于统一配置、显示和调参。

| 区间 | 语义 |
| --- | --- |
| 0—19 | 极低 |
| 20—39 | 较低 |
| 40—59 | 正常 |
| 60—79 | 较高 |
| 80—100 | 极高 |

基础数值默认 50。普通修正以 `±5—15` 为主，强事件或专属能力可达到 `±20—25`，最终结果统一钳制在合法范围。

### 2.1 概率原则

所有关键概率都保留失败空间：

| 场景 | 目标成功率 |
| --- | --- |
| 普通局势下的合理指挥 | 60%—75% |
| 队友性格与指令冲突 | 35%—55% |
| 高压、低信任、路线事故叠加 | 15%—40% |
| 强制命令 | 当下高成功，后续高代价 |

除特殊剧情或教程外，概率默认钳制在 `5%—95%`。游戏不提供绝对安全的指挥。

## 3. 玩家干员属性

玩家可选角色统一称为“干员”。干员名称参考现代战术游戏的语感，但不直接使用现实部队、商业作品干员名、技能名或阵营名。每名干员由“战术位 + 原创代号”组成，例如 `突击位·威风龙`。

基础战术位：

| 战术位 | 设计作用 | 命名语感 |
| --- | --- | --- |
| 突击位 | 打开正面局势、压制敌人、制造短期士气 | 动物、冲锋、破门、火力意象 |
| 支援位 | 治疗、救援、减压、修复信任 | 医疗包、信标、守护、稳定意象 |
| 侦察位 | 提前识别路线风险、伏击和隐藏收益 | 视野、夜行、标记、情报意象 |
| 工程位 | 控制物资、负重、撤离收益和事故补救 | 工具、背包、架设、修补意象 |
| 指挥位 | 强化队伍服从、协同和临场拍板 | 口令、哨声、战术板、权威意象 |

玩家属性分为“统筹、信息、行动、支援”四组。属性不是单纯战斗力，而是参与指令、事件、AI 服从、事故修正和资源管理。

### 3.1 统筹属性

| 属性 | 作用 |
| --- | --- |
| command 指挥 | 提高 AI 接受明确指令的意愿，影响队伍协同 |
| authority 威慑 | 提高强制命令、压制争执、快速拍板的效果 |
| empathy 共情 | 提高安抚、劝说、修复信任和识别情绪的效果 |

指挥解决“愿不愿意听”，威慑解决“现在必须听”，共情解决“之后还愿不愿意跟你干”。

### 3.2 信息属性

| 属性 | 作用 |
| --- | --- |
| insight 洞察 | 提高敌情、收益、陷阱和队友意向的预判准确度 |
| judgment 判断 | 降低错误情报、误判优势、被诱导深入的概率 |

洞察让玩家看到更多线索，判断让玩家少被错误线索骗。

### 3.3 行动属性

| 属性 | 作用 |
| --- | --- |
| combat 战斗 | 提高战斗、压制、突破、追击类事件贡献 |
| mobility 机动 | 提高撤离、转点、救援抵达、脱离包围的成功率 |

战斗强不等于局面稳定。高战斗角色更容易打开局面，也更容易把误判升级成事故。

### 3.4 支援属性

| 属性 | 作用 |
| --- | --- |
| medical 医疗 | 提高治疗、救援、伤势控制和倒地容错 |
| logistics 后勤 | 提高物资管理、负重控制、分配谈判和撤离收益 |

支援属性主要提高“局面变差后还能不能撑住”，不应直接抹平前期错误。

### 3.5 玩家局内派生状态

玩家在局内也会被消耗，而不是稳定站在棋盘外。

| 状态 | 范围 | 说明 |
| --- | --- | --- |
| health 生命 | 0—100 | 玩家倒地或死亡会导致失败或高代价撤离 |
| stamina 体力 | 0—100 | 连续战斗、救援、转点会消耗，低体力降低行动贡献 |
| focus 专注 | 0—100 | 影响情报判断、劝说和关键指令稳定度 |
| commandPressure 指挥压力 | 0—100 | 连续下达强指令、队友违抗、事故发生时上升 |
| injury 伤势 | 0—100 | 长期负面状态，治疗只能部分修复 |

玩家不是“上帝视角指挥官”。当指挥压力和伤势过高时，即使玩家干员数值优秀，也会出现误判、劝说失败和强制代价提高。

## 4. 首批玩家干员

### 4.1 支援位·白箱鸽

旧称：医生。

定位：通过治疗、减压、救援和关系修复提高整队容错。白箱鸽不是“奶妈无脑保命”，而是让已经变坏的局面多一次缓冲。

```text
command 50
authority 35
empathy 70
insight 55
judgment 60
combat 35
mobility 45
medical 80
logistics 50
```

初始被动“稳定救治”：

- 每局首次有队员倒地时，可将其恢复至 30 生命。
- 被救队员 `trust +8`，`panic -10`。
- 全队 `stress -5`。
- 玩家 `stamina -10`。

初始专属指令“战地处理”：

- 一名队员恢复 20 生命。
- 降低 12 压力和 8 恐慌。
- 本节点该成员不参与主要行动。
- 若该成员是高表现欲 AI，可能产生 `resentment +4`，认为自己被按住了。

策略代价：白箱鸽能修复失败，但不擅长用威慑压住队友，也无法让高风险进攻变得安全。

### 4.2 突击位·威风龙

旧称：突击手。

定位：快速解决战斗，通过压制和胜利建立短期凝聚力。威风龙强调“冲开局面”，但不是稳定答案；越敢打，误判时代价越大。

```text
command 50
authority 65
empathy 35
insight 45
judgment 45
combat 80
mobility 60
medical 25
logistics 35
```

初始被动“破门士气”：

- 使用进攻指令并成功时，`cohesion +6`。
- 若击退敌方或快速完成节点，参与者 `morale +8`。
- 失败时，`threat +8`、参与者 `stress +5`、玩家 `commandPressure +8`。

初始专属指令“压制突入”：

- 本节点队伍战斗效用 +15。
- 服从该指令的 AI 无法选择“稳守”。
- 若情报误判，本次事故严重度 +10。

策略代价：威风龙能把局势打穿，但会把错误判断放大成雪崩。

### 4.3 侦察位·夜眼狐

旧称：侦察员。

定位：提高信息质量，提前识别伏击、撤离窗口和队友偏离。夜眼狐让玩家更早看见风险，但不能替玩家解决已经爆炸的冲突。

```text
command 45
authority 35
empathy 50
insight 80
judgment 70
combat 45
mobility 65
medical 30
logistics 45
```

被动“先行观察”：

- 每个节点额外揭示一条隐藏风险或收益信息。
- 首次情报误判可以重新判断。
- 路线选择场景中，可提前识别一个节点的隐藏标签。

专属指令“标记路线”：

- 预览下一节点的类型、威胁区间和可能事故。
- 若选择该节点，全队初始压力 -5。
- 若未选择该节点，部分高纪律 AI `trust +3`，认为玩家尊重情报。

策略代价：夜眼狐擅长降低未知，但不擅长处理已经爆炸的正面冲突。

### 4.4 指挥位·铁哨

旧称：指挥官。

定位：提高服从和协同，通过强组织力压住混乱。铁哨能让队伍短时间“听口令”，但越压人，后续反弹越明显。

```text
command 80
authority 70
empathy 40
insight 55
judgment 60
combat 45
mobility 40
medical 30
logistics 50
```

被动“统一口径”：

- 凝聚力高于 60 时，指令影响 +8。
- 队友连续两次服从后，第三次指令的基础服从 -5，代表疲劳和逆反。
- 强制命令导致的信任损失增加 50%。

专属指令“最终决定”：

- 本节点全员执行同一指令。
- 每名被迫服从的 AI `trust -10`、`resentment +8`。
- 玩家 `commandPressure +12`。
- 每局一次。

策略代价：铁哨最能统筹队伍，但如果一直压人，会把后半局变成集体反弹。

### 4.5 工程位·背包熊

旧称：后勤员。

定位：控制物资分配、负重、贪婪压力和撤离收益。背包熊不是纯经济角色，而是负责让队伍“带着东西活着出去”。

```text
command 55
authority 45
empathy 55
insight 55
judgment 65
combat 30
mobility 45
medical 35
logistics 80
```

被动“公平分配”：

- 获得物资时 `greedPressure` 增量降低 25%。
- 队伍负重带来的撤离惩罚降低 20%。
- 全队撤离时物资价值 +10%。

专属指令“现场分配”：

- `greedPressure -15`。
- 物资欲最高的 AI `trust +5`。
- 其他未获益 AI 可能 `resentment +3`。
- 消耗当前物资价值的 5%。

策略代价：背包熊能让队伍“带着东西活着出去”，但正面战斗和急救能力较弱。

MVP 建议仅实现 `支援位·白箱鸽` 和 `突击位·威风龙`，其余干员在 Alpha 阶段加入。

## 5. AI 队友属性

AI 队友由四类数值组成：能力、驱动力、局内状态、关系状态。

### 5.1 能力属性

| 属性 | 作用 |
| --- | --- |
| combat 战斗 | 攻击、压制、追击等事件贡献 |
| support 支援 | 治疗、救援、安抚等事件贡献 |
| scouting 侦察 | 发现风险、识别伏击、修正情报 |
| carrying 携行 | 安全携带物资、降低负重惩罚 |
| discipline 纪律 | 执行计划、保持阵型、避免擅自行动 |
| improvisation 应变 | 事故发生时临场补救 |

纪律高的队友更听话，但不一定聪明；应变高的队友可能擅自救场，也可能擅自改计划。

### 5.2 驱动力

| 驱动力 | 高值倾向 | 低值倾向 |
| --- | --- | --- |
| lootDrive 物资欲 | 搜索、争抢、私藏、延迟撤离 | 放弃收益 |
| survivalDrive 生存欲 | 稳守、后撤、拒绝救援、保命优先 | 接受危险 |
| egoDrive 表现欲 | 追击、抢功、冒进、质疑保守命令 | 配合、不出头 |
| herdDrive 从众欲 | 跟随多数、受队友影响强 | 更独立 |
| revengeDrive 复仇欲 | 受伤或队友倒地后追击敌人 | 更容易止损 |
| greedTolerance 分配容忍 | 能接受少拿物资 | 容易因分配不满 |
| riskAppetite 风险偏好 | 高风险高收益行为效用提高 | 倾向安全选项 |
| blameTendency 甩锅倾向 | 失败后制造矛盾、降低凝聚力 | 更愿意共同承担 |

驱动力不是负面属性。高生存欲可能救全队，高表现欲可能破局，高从众欲可能让队伍稳定，也可能跟着错误的人一起乱冲。

### 5.3 AI 局内动态属性

| 属性 | 初始范围 | 变化来源 |
| --- | --- | --- |
| health 生命 | 100 | 受伤、治疗 |
| stamina 体力 | 60—100 | 战斗、搜索、转点、救援 |
| stress 压力 | 10—30 | 威胁、失败、安抚 |
| panic 恐慌 | 0—20 | 突发事故、倒地、包围 |
| trust 信任 | 35—65 | 指令结果、救援、分配 |
| resentment 怨气 | 0—20 | 强制、分配不公、被牺牲 |
| morale 士气 | 40—70 | 胜利、救援、顺利撤离推进 |
| suppression 压制 | 0—100 | 火力压制、受伤、被包围 |
| personalLoot 个人物资 | 0 | 搜索、争抢、分享 |
| fatigue 疲劳 | 0—40 | 连续行动、负重、伤势 |

这些属性共同决定 AI 的当前状态。一个队友可能“信任玩家但很恐慌”，也可能“士气很高但怨气很重”。系统不应把复杂状态压成单一好感度。

### 5.4 队友之间的关系

除 AI 对玩家的 `trust` 外，还存在轻量队友关系：

| 属性 | 作用 |
| --- | --- |
| pairTrust 双人信赖 | 影响互相救援、掩护和跟随 |
| pairConflict 双人矛盾 | 影响甩锅、抢物资和拒绝协同 |

MVP 可不显示完整关系网，只在事件反馈中展示关键矛盾，例如：

> 阿强拒绝掩护小刀：之前物资分配冲突仍未解决，且当前压力过高。

## 6. 全队局内状态

全队状态描述“这一局现在有多容易炸”。

```text
RunState
  threat: 0—100          # 敌人与环境危险
  cohesion: 0—100        # 队伍整体凝聚力
  greedPressure: 0—100   # 物资引发的争抢压力
  exposure: 0—100        # 暴露程度，越高越容易遭遇伏击或追击
  noise: 0—100           # 噪音，影响敌人增援和撤离难度
  load: 0—100            # 队伍负重，影响机动和撤离
  evacWindow: 0—100      # 撤离窗口稳定度，越低越难安全撤离
  accidentHeat: 0—100    # 事故热度，越高越容易触发连锁事件
  lootValue: int         # 已获得但尚未带出的收益
  nodeIndex: int
```

局内随机性的主要来源不是纯骰子，而是这些状态互相叠加后把事件推向更糟或更好的分支。

## 7. AI 队友行动模板

AI 队友不使用固定英雄名，而使用“行动模板 + 随机昵称”的方式生成。模板定义属性范围、驱动力倾向和特殊规则；昵称只用于记忆与事件反馈，避免玩家把随机队友误解成可长期培养的固定角色。

生成示例：

```text
行动模板：潜搜者
随机昵称：阿强
显示名：阿强 / 潜搜者
```

生成时在模板范围内加入 `±10` 浮动，并随机附加一个小特质。

| 模板 | 驱动力倾向 | 特殊规则 |
| --- | --- | --- |
| 潜搜者 | 高物资欲、中生存欲、低表现欲 | 独自搜索的收益和风险都提高 |
| 猛攻者 | 低生存欲、高表现欲、高风险偏好 | 敌情被判断为弱时更容易追击 |
| 保命派 | 高生存欲、低风险偏好 | 生命低于 50 时服从进攻大幅下降 |
| 协同者 | 中等驱动力、高从众欲 | 高凝聚力时服从与支援提高 |
| 夺令者 | 中表现欲、中纪律、低服从耐性 | 连续服从后可能要求改变计划 |
| 掷骰手 | 高物资欲、高表现欲、高风险偏好 | 高风险高收益行为效用提高 |
| 甩锅者 | 高甩锅倾向、中生存欲 | 失败后更容易引发队友矛盾 |
| 老鸟 | 高纪律、高判断、低情绪波动 | 压力扰动较低，但讨厌无意义冒险 |
| 新丁 | 低纪律、低判断、高从众欲 | 容易跟随附近队友，受恐慌影响更强 |
| 孤行者 | 低从众欲、高应变 | 更容易擅自行动，也更可能单人补救事故 |

### 7.1 随机小特质

每名 AI 生成时可附带 0—1 个小特质：

| 特质 | 效果 |
| --- | --- |
| 碎嘴 | 压力上升时更容易暴露队伍意图，`noise +` |
| 快手 | 搜索和追击更强，但误触事故概率提高 |
| 护队 | 更愿意救援 pairTrust 高的队友 |
| 记账 | 怨气下降更慢 |
| 财眼 | 高价值物资出现时物资欲临时提高 |
| 稳心 | 恐慌上升较慢，但表现欲更低 |
| 犟骨 | 低生命时仍可能拒绝撤离 |

小特质用于制造可记忆的队友，而不是堆数值。

## 8. 服从与行为效用

### 8.1 指令接受分

```text
指令接受分 =
35
+ player.command × 0.22
+ player.authority × 0.12
+ teammate.trust × 0.20
+ run.cohesion × 0.14
+ teammate.discipline × 0.10
- teammate.stress × 0.16
- teammate.panic × 0.18
- teammate.resentment × 0.20
- 玩家指令与个人驱动力冲突 × 0.25
- player.commandPressure × 0.08
+ 卡牌、角色、事件修正
+ random(-10, 10)
```

达到 60 时，AI 倾向执行玩家指令；未达到时，AI 会在个人候选行为中选择效用最高的一项。

### 8.2 执行稳定度

服从不代表行动一定成功。服从后还需要计算执行稳定度：

```text
执行稳定度 =
行动基础值
+ 对应能力贡献
+ player.judgment × 0.10
+ run.cohesion × 0.10
+ teammate.morale × 0.08
- run.threat × 0.15
- run.exposure × 0.10
- teammate.fatigue × 0.12
- teammate.suppression × 0.15
- 情报误差
+ random(-12, 12)
```

这能制造一种重要体验：队友听了你的，但局势太差，还是执行失败。

### 8.3 个人行为效用

以“搜索”为例：

```text
searchUtility =
20
+ lootDrive × 0.38
+ expectedLoot × 0.25
+ riskAppetite × 0.12
- threat × survivalDrive / 220
- load × 0.10
+ personalityModifier
+ random(-10, 10)
```

以“擅自追击”为例：

```text
chaseUtility =
15
+ egoDrive × 0.35
+ revengeDrive × 0.25
+ combat × 0.18
- survivalDrive × 0.18
- suppression × 0.20
+ enemyWeaknessSignal
+ random(-12, 12)
```

以“拒绝撤离”为例：

```text
refuseEvacUtility =
10
+ lootDrive × 0.25
+ egoDrive × 0.18
+ personalLootOpportunity
- survivalDrive × 0.12
- trust × 0.15
+ resentment × 0.20
+ random(-10, 10)
```

AI 不应只有“听话/不听话”两种结果，而应有多种偏离方式：慢半拍、改目标、独自行动、拉队友一起行动、嘴上答应但执行保守。

### 8.4 犹豫、劝说与强制

若 AI 的前两名行为效用差值小于 8，进入犹豫状态。玩家可选择：

| 操作 | 主要属性 | 效果 | 代价 |
| --- | --- | --- | --- |
| 劝说 | empathy / command | 提高接受分 | 消耗 focus，失败增加压力 |
| 施压 | authority / command | 大幅提高当下服从 | 增加怨气和指挥压力 |
| 让步 | logistics / empathy | 降低冲突或贪婪压力 | 损失收益或节奏 |
| 放任 | 无 | 不消耗资源 | AI 自行选择 |

强制机制：

```text
强制成功：立即执行
信任损失：5 + 冲突驱动力 / 10
怨气增加：4 + teammate.resentment / 20
玩家指挥压力：+8
全队凝聚力：-3，若结果成功则返还 2
```

强制命令是救急工具，不是长期管理工具。

## 9. 随机性系统

随机性分为三类，避免所有变化都像纯 RNG。

### 9.1 开局随机

- 玩家匹配到的 AI 人格、能力和驱动力随机。
- 队友之间可能预设轻量关系：熟人、互相看不惯、曾经合作、单方面崇拜。
- 开局资源、路线主题和撤离规则随机。

### 9.2 路线随机

- 路线节点类型、事件标签、隐藏风险和奖励随机。
- 中间节点可以分叉与汇合，玩家不能完整预知所有后果。
- 同一事件在不同威胁、负重、队友状态下会进入不同分支。

### 9.3 局内扰动

```text
随机扰动范围 =
基础随机值
+ threat / 20
+ accidentHeat / 20
+ 平均压力 / 25
- player.judgment / 30
- run.cohesion / 30
```

局势越乱，骰子越大；玩家判断越好、队伍越稳，骰子越小。但骰子永远不会归零。

## 10. 局内成长

局内成长不是稳定变强，而是角色在本局中形成临时状态、关系和行为倾向。

### 10.1 信任阶段

| 信任 | 状态 | 效果 |
| --- | --- | --- |
| 0—19 | 敌对 | 更可能拒绝、私藏、甩锅或独自撤离 |
| 20—39 | 怀疑 | 指令接受 -8，劝说难度提高 |
| 40—59 | 普通 | 无修正 |
| 60—79 | 信赖 | 指令接受 +8，救援倾向提高 |
| 80—100 | 托付 | 每局可触发一次无条件协同行为 |

### 10.2 怨气阶段

| 怨气 | 状态 | 效果 |
| --- | --- | --- |
| 0—19 | 无明显不满 | 无修正 |
| 20—39 | 不爽 | 强制和分配相关指令接受下降 |
| 40—69 | 抵触 | 更容易阴阳怪气、拖延、改动作 |
| 70—100 | 爆发 | 触发争吵、独走、拒绝救援或抢物资事件 |

信任和怨气可以同时存在。一个队友可能信任你的判断，但很不满你刚才牺牲了他的收益。

### 10.3 压力与恐慌阶段

| 压力 | 状态 | 效果 |
| --- | --- | --- |
| 0—29 | 冷静 | 行为稳定 |
| 30—59 | 紧张 | 随机扰动略增 |
| 60—79 | 失衡 | 性格驱动力 +20% |
| 80—100 | 崩溃 | 触发人格相关极端行为 |

| 恐慌 | 状态 | 效果 |
| --- | --- | --- |
| 0—19 | 可控 | 无明显影响 |
| 20—49 | 慌乱 | 执行稳定度下降 |
| 50—79 | 失措 | 可能误判、漏执行、跟随错误目标 |
| 80—100 | 崩盘 | 可能逃跑、乱开火、放弃队友 |

压力是长期积累，恐慌是短期爆点。支援位·白箱鸽、协同者队友和高共情干员更擅长处理恐慌。

### 10.4 士气阶段

| 士气 | 状态 | 效果 |
| --- | --- | --- |
| 0—29 | 低落 | 支援和服从下降 |
| 30—59 | 普通 | 无修正 |
| 60—79 | 高昂 | 战斗和救援倾向提高 |
| 80—100 | 上头 | 战斗变强，但冒进和追击概率提高 |

士气不是纯正面状态。过高士气会让猛攻者和掷骰手更危险。

### 10.5 临时专长

事件可以给予三选一的本局增益，例如：

| 专长 | 效果 | 代价 |
| --- | --- | --- |
| 背水一战 | 生命低于 40 时战斗 +10 | 恐慌增长 +10% |
| 搜刮直觉 | 搜索收益 +15% | 物资欲影响 +10 |
| 同生共死 | 救援成功时双方信任 +5 | 救援失败时双方压力 +8 |
| 打顺风局 | 连续成功后士气收益提高 | 失败时士气损失提高 |
| 老练撤退 | 撤离行动稳定度 +10 | 追击收益 -10% |
| 记住这笔账 | 被强制后短期服从 +5 | 怨气不会自然下降 |

每名角色最多持有 2 个临时专长。超过上限时必须替换，避免后期状态难以阅读。

### 10.6 局内角色能量

玩家干员可拥有本局资源，用于制造职业差异。

| 角色 | 局内资源 | 获得方式 | 用途 |
| --- | --- | --- | --- |
| 支援位·白箱鸽 | 医疗包扎 | 完成救援、低伤通过节点 | 治疗、减恐慌、稳定伤势 |
| 突击位·威风龙 | 突破节奏 | 快速解决战斗、压制成功 | 强化进攻、降低敌方行动 |
| 侦察位·夜眼狐 | 情报点 | 识别隐藏风险、避开伏击 | 揭示路线、修正误判 |
| 指挥位·铁哨 | 指挥权威 | 连续协同成功、队伍服从 | 强化命令、短期统一行动 |
| 工程位·背包熊 | 物资调度 | 合理分配、低损耗撤离 | 降低负重、平息争抢 |

角色能量应鼓励玩家使用角色特色，而不是变成通用蓝条。

## 11. 事故系统

事故是高随机体验的核心。事故不是单独随机事件，而是状态累积后的爆发。

### 11.1 事故热度

```text
事故热度增加 =
节点基础危险
+ threat / 10
+ exposure / 12
+ noise / 15
+ 平均压力 / 15
+ greedPressure / 18
- player.judgment / 20
- scouting总贡献 / 25
```

事故热度越高，事件越容易从普通分支滑向复杂分支。

### 11.2 事故类型

| 事故 | 触发倾向 | 结果 |
| --- | --- | --- |
| 伏击 | 高暴露、高噪音、侦察失败 | 受伤、压制、撤离窗口下降 |
| 争抢物资 | 高物资压力、高物资欲 | 怨气、私藏、队友冲突 |
| 擅自追击 | 高表现欲、高复仇欲、高士气 | 分队、受伤、额外收益 |
| 恐慌撤退 | 高恐慌、高生存欲 | 队伍分散、撤离提前 |
| 误判情报 | 低判断、高隐藏风险 | 指令效果反转或削弱 |
| 甩锅争吵 | 高甩锅倾向、连续失败 | 凝聚力下降、信任变化 |
| 负重拖累 | 高负重、低体力 | 撤离失败率提高、抛弃物资 |

每次事故都应生成 1—3 条可读原因，避免玩家觉得系统在暗箱惩罚。

## 12. 局外成长

局外成长以横向解锁为主，不提供无限叠加的基础强度。

### 12.1 账号等级

用途：按节奏解锁系统内容。

- 完成对局获得行动经验。
- 成功撤离提供主要奖励。
- 失败提供约 30% 基础经验。
- 高质量失败可额外提供“事故记录”奖励，例如识别伏击但撤离失败。
- 账号等级解锁新角色、新指令、新事件、新路线主题和新 AI 人格。

账号等级不直接增加永久基础伤害。

### 12.2 角色熟练度

每个角色独立积累，建议 10 级封顶。

| 等级 | 解锁 |
| --- | --- |
| 1 | 基础角色和默认能力 |
| 2 | 第一张角色指令 |
| 3 | 第二张角色指令 |
| 4 | 第一组二选一专长 |
| 5 | 角色事件与语音/档案 |
| 6 | 第三张角色指令或指令变体 |
| 7 | 挑战修正 |
| 8 | 第二组二选一专长 |
| 9 | 高难专属挑战 |
| 10 | 荣誉奖励、外观或称号 |

角色熟练度的重点是改变打法，而不是提高通用胜率。

### 12.3 角色专长示例

支援位·白箱鸽专长：

- 急救专家：治疗量提高，但减压效果降低。
- 心理干预：减压和信任效果提高，但治疗量降低。
- 战场分诊：多人轻伤时效率提高，但单人重伤治疗较弱。

突击位·威风龙专长：

- 破门手：开局前两次战斗压制提高，但噪音增加。
- 稳准狠：战斗成功率更稳定，但士气收益降低。
- 越打越凶：连续成功后战斗提高，但失败事故热度提高。

侦察位·夜眼狐专长：

- 风险地图：路线隐藏标签识别提高，但战斗贡献下降。
- 临场预警：伏击事故严重度降低，但每局次数有限。

指挥位·铁哨专长：

- 冷面权威：强制命令代价降低，但共情类劝说变弱。
- 共同目标：高凝聚力时全队服从提高，但低凝聚力时效果更差。

工程位·背包熊专长：

- 精打细算：物资价值提高，但分配争执更频繁。
- 轻装撤离：负重惩罚降低，但最大收益上限降低。

### 12.4 指令熟练度

指令也可以解锁变体，但不无限升级。

| 指令 | 变体示例 |
| --- | --- |
| 集火 | 稳定集火：伤害较低但压制更稳 |
| 搜索 | 快速搜索：收益较低但威胁增长少 |
| 撤离 | 分批撤离：更安全但可能损失物资 |
| 救援 | 冒险救援：成功收益高，失败事故重 |
| 劝说 | 私下劝说：单人效果强，但消耗时间 |

玩家构筑的是“指挥风格”，不是数值碾压。

### 12.5 资源解锁

带出的物资转化为“行动资金”，用于解锁横向内容。

允许购买：

- 新指令。
- 开局消耗品。
- 角色专长预设栏。
- 路线主题。
- AI 人格池扩展。
- 外观和档案。

不允许购买：

- 无限叠加属性。
- 永久大幅伤害加成。
- 跳过 AI 偏离的绝对控制权。

## 13. 平衡规则

- 永久属性加成总量不超过基础值的 10%。
- 新角色必须同时有强项、代价和独特决策。
- 新属性只有在至少三个系统使用时才能加入。
- 同类加成统一使用加法，最终倍率统一在末端计算。
- 所有概率默认钳制在 5%—95%。
- 局内临时效果默认不超过 2 层。
- 队伍状态越糟，随机扰动越大；但原因必须可解释。
- 强制命令解决当下问题，制造后续问题。
- 失败应产生故事、经验或解锁线索，不能只是扣奖励。

## 14. 初始平衡基线

| 项目 | 建议值 |
| --- | --- |
| 普通 AI 初始信任 | 50 ± 15 |
| 普通 AI 初始压力 | 20 ± 10 |
| 普通 AI 初始怨气 | 0—15 |
| 普通 AI 初始士气 | 50 ± 15 |
| 初始凝聚力 | 55 |
| 初始威胁 | 15 |
| 初始暴露 | 10 |
| 初始噪音 | 0 |
| 普通节点威胁增量 | 5—12 |
| 普通节点事故热度增量 | 8—18 |
| 战斗失败压力增量 | 12—20 |
| 指令成功目标概率 | 约 60%—70% |
| 明显性格冲突下成功概率 | 约 35%—50% |
| 高压事故局面成功概率 | 约 15%—40% |
| 单局角色能力使用次数 | 1—4 次 |
| 单局平均事故次数 | 2—4 次 |
| 单局严重事故次数 | 0—2 次 |

平衡时优先记录以下数据：

1. 玩家下达指令次数。
2. AI 服从、犹豫、偏离、强制次数。
3. 事故触发来源。
4. 队友怨气和信任变化。
5. 玩家主动撤离比例。
6. 失败时玩家能否复述原因。

如果玩家觉得“我完全管不了”，应优先加强信息反馈和局部补救手段。
如果玩家觉得“我总能稳定控住”，应提高事故热度、队友冲突和资源诱惑。

## 15. MVP 落地范围

虽然本文档定义了完整方向，MVP 不需要一次性实现全部系统。

MVP 必须实现：

- 玩家干员：支援位·白箱鸽、突击位·威风龙。
- 玩家属性：command、authority、empathy、insight、judgment、combat、medical。
- AI 属性：combat、support、discipline、improvisation。
- AI 驱动力：物资欲、生存欲、表现欲、从众欲。
- AI 状态：生命、压力、恐慌、信任、怨气、士气、个人物资。
- 全队状态：威胁、凝聚力、贪婪压力、暴露、事故热度、物资价值。
- 服从、犹豫、偏离、强制。
- 至少 6 种 AI 行动模板。
- 至少 5 种事故类型。

MVP 暂不实现：

- 完整队友关系网。
- 指令熟练度。
- 所有角色专长。
- 复杂伤势部位。
- 永久队友养成。
- 多层装备词条。

第一版目标不是系统完整，而是验证：玩家是否能感受到“我有办法影响这支队伍，但这支队伍真的不是我的傀儡”。
