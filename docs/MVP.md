# 《野排模拟器》MVP 设计文档

> 文档版本：0.5  
> 项目阶段：最小可玩原型  
> MVP 场景范围：路线选择场景、事件/互坑场景  
> 关联文档：[游戏设计总览](GAME_DESIGN.md) · [角色数值与成长](CHARACTER_AND_PROGRESSION.md) · [场景与界面流程](SCENE_DESIGN.md) · [技术架构](SYSTEM_ARCHITECTURE.md)

## 1. MVP 一句话定义

玩家扮演三人野排小队中的一员，在随机路线中和两个高度自私、会单走、会抢包、会卖人的 AI 队友一起行动，通过合作、欺骗、抢包、丢包和撤离选择，尽量带着物资活着出去。

MVP 的核心不是验证“玩家能否指挥 AI”，而是验证三个问题：

1. 两名 AI 主动作恶、单走和互坑是否有趣且可解释。
2. 玩家作恶行为是否有明确收益、代价和黑色喜剧效果。
3. 背包归属与撤离姿态是否能支撑风险收益循环。

## 2. MVP 范围原则

本版本主动压缩局外系统，只验证局内互坑撤离闭环。

### 2.1 只做两个核心场景

| 场景 | 必须实现 | 主要职责 |
| --- | --- | --- |
| 路线选择场景 | 是 | 展示随机路线，从可达节点中选择下一事件 |
| 事件/互坑场景 | 是 | 展示事件、AI 意图、玩家行动、互坑结算和撤离 |

MVP 的实际循环是：

```text
路线选择
→ 选择可达节点
→ 进入事件/互坑场景
→ 两名 AI 生成意图
→ 玩家选择自己的行动和一句简单喊话
→ 结算合作、单走、抢包、倒地、丢包、撤离
→ 返回路线选择或进入结算
```

### 2.2 暂不作为 MVP 场景

以下内容可以用临时数据、调试按钮、弹窗或后续版本实现：

- 主菜单。
- 完整行动准备。
- 完整角色选择。
- 指令库或卡牌构筑。
- 完整局外大厅。
- 完整对局结算场景。
- 事故图鉴。
- 教程。
- 永久队友养成。

原型可以启动后直接进入路线选择场景，并使用默认玩家角色、默认起始背包和随机 AI 队友。

## 3. MVP 要验证的问题

MVP 只验证以下假设：

1. 玩家能理解自己不是队长，只是三人之一。
2. 玩家能看懂 AI 大概想干什么，并预期它们可能作恶。
3. AI 单走、抢包、丢包、卖人不会显得像纯随机惩罚。
4. 玩家可以主动做坏事，并感到这是有效玩法而不是装饰按钮。
5. 物资绑定到个人背包后，玩家会在“多拿一点”和“现在跑路”之间犹豫。
6. 一局 5—8 个节点内至少出现一次队伍分裂、一次抢包或卖人、一次撤离取舍。

若玩家觉得“我只是被系统坑”，优先加强事前线索和反制选择。若玩家觉得“作恶不划算”，优先提高抢包、卖人、丢包撤离的短期收益。

## 4. 单局最小流程

```text
生成玩家、两名 AI 队友、初始 RunState
→ 生成随机路线图
→ 进入路线选择场景
→ 玩家选择下一节点
→ 进入事件/互坑场景
→ 展示事件情报、物资机会、撤离窗口和队友意图
→ 玩家选择行动与喊话
→ 结算三人行动
→ 更新生命、背包、位置、压力、仇恨、威胁和撤离窗口
→ 返回路线选择场景
→ 玩家撤离、死亡、被困或路线结束后结算
```

## 5. 核心对象

### 5.1 角色位置

MVP 不做复杂地图，但必须有轻量位置状态。

| 状态 | 说明 |
| --- | --- |
| 同行 | 与玩家在同一区域，可直接合作或冲突 |
| 邻近 | 可以支援或救援，但有风险和延迟 |
| 单走 | 独自在别处行动，收益和风险都提高 |
| 倒地 | 可被救、被搜包或被抛弃 |
| 撤离中 | 正在撤离，可能被打断或抢包 |
| 已撤离 | 该角色本局结束 |
| 死亡/失踪 | 物资可能掉落或丢失 |

队伍分裂是 MVP 必须验证的核心状态。事件结算后，三人可以形成：

- 三人同行。
- 玩家 + 一名 AI，另一名 AI 单走。
- 玩家单走。
- 两名 AI 结伴，玩家落单。
- 有人倒地，其他人选择救援、搜包或撤离。

### 5.2 个人背包

物资绑定到人，而不是只存在于全队总数。

```text
ActorInventory
  light_loot_value
  heavy_loot_value
  rare_loot_value
  mission_item_count
  weight
  stolen_from_actor_ids[]
```

MVP 简化规则：

- 每名角色都有独立背包价值和负重。
- 只有成功撤离角色身上的物资才算带出。
- 倒地角色的背包可以被玩家或 AI 搜走。
- 丢包会减少背包价值和负重，提高撤离或逃跑稳定度。
- 抢包会提高收益，但提高被报复、被追击或内斗概率。

### 5.3 关系与仇恨

MVP 不做完整关系网，只保留关键数值。

```text
ActorRelationToPlayer
  trust: 0—100
  suspicion: 0—100
  grudge: 0—100
```

```text
RunState
  threat: 0—100
  exposure: 0—100
  chaos: 0—100
  evac_window: 0—100
  player_infamy: 0—100
```

含义：

- `trust`：AI 是否相信玩家喊话和承诺。
- `suspicion`：AI 是否怀疑玩家要抢、骗、卖。
- `grudge`：AI 是否倾向报复玩家。
- `chaos`：局势混乱程度，越高越容易出现极端行为。
- `player_infamy`：玩家本局恶名，越高越容易被 AI 防备或反咬。

## 6. 路线选择场景

路线选择场景是 MVP 的战略层。玩家只做一个主要决策：下一事件去哪。

建议路径：`res://scenes/match/route_map.tscn`

### 6.1 场景职责

- 展示随机生成的分层路线图。
- 标记当前节点、已完成节点、可达节点和锁定节点。
- 展示每个可达节点的基础预览信息。
- 展示队友对节点的兴趣或抗拒。
- 允许玩家选择一个可达节点进入事件/互坑场景。

路线选择场景不直接结算战斗和互坑，只负责选择下一事件。

### 6.2 路线结构

MVP 使用 5 层路线：

```text
起点 → 第 1 层 → 第 2 层 → 第 3 层 → 第 4 层 → 撤离层
          2—4       2—4       2—4       2—4       1—2
```

规则：

1. 第 1—4 层每层生成 `2—4` 个事件节点。
2. 第 5 层生成 `1—2` 个撤离或高风险终点节点。
3. 起点连接第 1 层所有节点。
4. 普通节点连接下一层 `1—2` 个节点。
5. 少数分叉节点可连接 3 个节点。
6. 每个非终点节点至少有一条出边。
7. 每个生成节点至少能从起点抵达。
8. 地图整体至少有 2 次有效分叉。
9. 同一路径上不能连续出现 3 个相同类型节点。

### 6.3 节点类型

MVP 保留 6 类节点。

| 类型 | 作用 | 示例 |
| --- | --- | --- |
| 战斗 | 消耗状态，制造倒地和抢包机会 | 遭遇敌方小队 |
| 搜索 | 获得物资，提高单走和抢包诱惑 | 废弃仓库、空投箱 |
| 争执 | 队内分配、甩锅和互相怀疑 | 稀有物资归属冲突 |
| 事故 | 已有问题爆发 | 队友把敌人引过来 |
| 撤离 | 尝试结束本局 | 临时撤离点、最终撤离 |
| 未知 | 信息模糊，风险和收益不明 | 枪声方向、陌生信号 |

### 6.4 节点预览信息

每个可达节点至少显示：

- 节点名称或模糊名称。
- 节点类型。
- 预计威胁：低 / 中 / 高。
- 预计收益：低 / 中 / 高。
- 特殊标签：如“高物资”“易分裂”“撤离窗口”“疑似伏击”。
- 两名 AI 的态度预测：想去、抗拒、可能单走、看不准。

示例：

```text
空投箱落点
类型：搜索
威胁：高
收益：高
标签：高价值 / 易抢包 / 可能伏击

阿强：非常想去，可能单走
老周：想绕开，背包已经很重
```

## 7. 事件/互坑场景

事件/互坑场景是 MVP 的执行层。所有战斗、搜索、争执、事故和撤离节点都进入这个场景处理。

建议路径：`res://scenes/match/encounter.tscn`

### 7.1 场景职责

- 展示当前事件的文本、背景和已知情报。
- 展示玩家与两名 AI 的生命、背包、位置和主要意图。
- 展示玩家可选行动。
- 允许玩家选择一句简单喊话。
- 结算三人行动、队伍分裂、抢包、倒地、丢包和撤离。
- 展示结果原因与状态变化。
- 返回路线选择场景，或进入撤离/失败结果。

### 7.2 场景阶段

```text
事件进入
→ 情报展示
→ AI 意图预览
→ 玩家选择行动
→ 玩家选择可选喊话
→ 三人行动结算
→ 结果展示
→ 返回路线 / 撤离 / 失败
```

MVP 不做复杂卡牌和指令构筑。玩家每回合从固定行动列表中选择一个主要行动。

### 7.3 玩家行动

MVP 固定 10 个行动。

| 行动 | 主要效果 | 主要风险 |
| --- | --- | --- |
| 跟随队友 | 降低落单风险 | 被队友带进坏局面 |
| 搜索物资 | 获得个人背包物资 | 暴露、负重、被盯上 |
| 稳住观察 | 降低事故并提高意图判断 | 收益较低 |
| 正面压制 | 提高战斗处理能力 | 受伤、噪音、队友误判 |
| 救援队友 | 恢复倒地或受伤队友 | 被伏击、被卖、错过撤离 |
| 搜队友包 | 获得倒地或死亡队友物资 | 恶名、仇恨上升 |
| 抢包 | 夺取附近队友部分物资 | 可能立刻引发内斗 |
| 丢包 | 降低负重，提高逃跑和撤离 | 损失收益 |
| 引怪转移 | 把威胁推向别人 | 可能反噬，恶名上升 |
| 撤离 | 推进个人撤离或结束本局 | 可能被打断，带货受限 |

### 7.4 简单喊话

玩家行动之外，可以选择一句喊话。喊话只影响 AI 判断，不保证生效。

| 喊话 | 作用 | 可能反噬 |
| --- | --- | --- |
| 跟我 | 提高附近 AI 跟随倾向 | 低信任时无效 |
| 撤了 | 提高撤离倾向 | 高物资欲 AI 可能拒绝 |
| 别舔了 | 降低搜索倾向 | 高物资欲 AI 怨气上升 |
| 我去救 | 提高倒地队友信任 | 不兑现会大幅加仇恨 |
| 你顶一下 | 推队友承担风险 | 被识破会增加怀疑 |
| 包归你 | 诱导物资欲 AI 配合 | 不兑现会引发报复 |
| 我断后 | 让队友更愿意撤 | 玩家先跑会增加恶名 |

MVP 可以先实现喊话的数值修正和事后背刺判定，不需要语音或复杂对话。

### 7.5 AI 意图

AI 每个事件从候选意图中选择一个。

| 意图 | 典型触发 |
| --- | --- |
| 跟随玩家 | 信任高、风险中等、自己背包轻 |
| 跟随另一名 AI | 从众欲高、玩家恶名高、另一 AI 更有吸引力 |
| 单走搜索 | 物资欲高、高价值节点、威胁可接受 |
| 正面开打 | 表现欲高、敌人显弱、士气高 |
| 稳住不动 | 生存欲高、生命低、威胁高 |
| 救援 | 信任高、队友倒地、支援倾向高 |
| 搜包 | 队友倒地、物资欲高、玩家离远 |
| 抢包 | 目标背包高价值、仇恨高、混乱高 |
| 丢包逃跑 | 生命低、撤离窗口低、负重高 |
| 提前撤离 | 背包价值高、信任低、生存欲高 |
| 引怪甩锅 | 甩锅倾向高、压力高、自己被追 |

AI 意图反馈需要展示 1—2 条主要原因。例如：

```text
阿强可能单走搜空投：
+ 空投价值很高
+ 物资欲极高
- 当前威胁偏高
```

## 8. 互坑与事故系统

MVP 至少实现 7 种互坑事故。

| 事故 | 触发倾向 | 结果 |
| --- | --- | --- |
| 单走引战 | AI 单走、高威胁、搜索失败 | 队伍分裂，主战区可能少人 |
| 抢包内斗 | 背包高价值、仇恨或贪婪高 | 伤害、物资转移、关系恶化 |
| 假救援 | 倒地队友有高价值包 | 搜包代替救援，倒地者仇恨上升 |
| 丢包跑路 | 低生命、高负重、撤离窗口低 | 角色轻装撤，物资损失 |
| 引怪卖人 | 压力高、甩锅倾向高 | 威胁转移，目标陷入危险 |
| 提前撤离 | 背包高价值、生存欲高 | 该角色退出，小队火力下降 |
| 甩锅争吵 | 连续失败、高恶名或高仇恨 | 混乱上升，下一次互坑概率提高 |

事故热度公式：

```text
chaos 增加 =
节点基础危险
+ threat / 10
+ exposure / 12
+ 平均压力 / 15
+ 队伍分裂修正
+ 高价值背包数量 × 4
+ player_infamy / 12
```

事故不是每次都触发，但 `chaos` 越高，AI 越容易做极端行为。

## 9. 背包、丢包与抢包

### 9.1 背包价值

MVP 只区分三种物资：

| 物资 | 特点 |
| --- | --- |
| 轻物资 | 价值低，负重低 |
| 重物资 | 价值中高，影响行动 |
| 稀有物 | 价值高，显著吸引抢包 |

### 9.2 丢包

玩家或 AI 可在危险时丢包。

```text
丢包效果：
- 背包价值减少
- weight 降低
- 撤离稳定度提高
- 被抢包概率降低
- 若丢弃任务物品，可能导致目标失败
```

### 9.3 抢包

抢包可以针对：

- 倒地角色。
- 死亡/失踪角色。
- 同行且背包很肥的队友。
- 撤离中的队友。

抢包结算：

```text
抢包成功率 =
基础值
+ 玩家机动/近身优势
+ 目标倒地或混乱修正
+ chaos 修正
- 目标警觉
- 目标战斗能力
- 附近第三人干预
```

抢包后果：

- 玩家获得物资。
- 玩家恶名上升。
- 被抢者若存活，仇恨大幅上升。
- 另一名 AI 可能跟抢、阻止或趁乱跑路。

## 10. 撤离系统

MVP 必须实现多种撤离姿态。

| 姿态 | 条件 | 结果 |
| --- | --- | --- |
| 独自轻装撤 | 玩家在撤离节点或撤离窗口，负重较低 | 成功率高，收益低 |
| 独自带货撤 | 玩家背包有价值，仍选择撤离 | 成功率中，收益中高 |
| 丢包撤 | 先丢包再撤 | 成功率高，收益大幅下降 |
| 带队友撤 | 至少一名 AI 同行并愿意撤 | 收益较高，但成功率受队伍状态影响 |
| 抢包撤 | 抢包后立刻撤 | 收益高，风险高，恶名高 |
| 被队友抢撤 | AI 抢走物资并撤 | 玩家损失收益，形成事故记录 |

撤离窗口：

```text
evac_window: 0—100
```

- 每完成节点后下降。
- 高威胁、噪音、混乱会额外下降。
- 到达 0 后撤离更难或只能轻装撤。
- 部分节点提供临时撤离机会。

## 11. MVP 玩家角色

MVP 只提供两个默认角色。角色不是队长职业，而是互坑风格。

### 突击位·威风龙

定位：擅长正面压制和抢节奏，能靠战斗制造抢包或撤离机会。

```text
combat 80
mobility 60
deception 35
support 25
carry 35
nerve 55
```

被动“破门上头”：

- 正面压制成功时，敌方压力下降，玩家可获得一次额外撤离或搜索窗口。
- 失败时，`threat +8`、`chaos +10`。

专属行动“压过去”：

- 本事件战斗成功率提高。
- 附近 AI 更容易被带动进攻。
- 若敌情误判，队伍更容易分裂或倒地。

### 支援位·白箱鸽

定位：擅长救援和装好人，能靠救人建立信任，也能利用倒地局面接近肥包。

```text
combat 35
mobility 45
deception 55
support 80
carry 45
nerve 60
```

被动“可信外表”：

- 第一次承诺救援时，AI 更容易相信玩家。
- 救援成功后，被救者 `trust +12`。
- 若喊了“我去救”却搜包或撤离，`player_infamy +20`，目标若存活则 `grudge +25`。

专属行动“战地处理”：

- 救起倒地队友并恢复少量生命。
- 也可以选择“处理背包”：更隐蔽地拿走倒地者部分物资，但救援效果降低。

## 12. MVP AI 人格

至少提供 6 种 AI 人格模板。

| 人格 | 倾向 | 作恶特点 |
| --- | --- | --- |
| 潜搜者 | 高物资欲、中生存欲 | 容易单走舔箱 |
| 猛攻者 | 高表现欲、低生存欲 | 容易开打、追击、把队伍拖进战斗 |
| 保命派 | 高生存欲、低风险偏好 | 容易丢包、提前撤离 |
| 赌徒 | 高风险偏好、高物资欲 | 喜欢高风险高收益，容易抢包 |
| 甩锅者 | 高甩锅、高压力反应 | 容易引怪、争吵、卖人 |
| 假好人 | 表面支援、实际贪 | 倒地时可能假救援真搜包 |

每名 AI 生成时在模板范围内加入浮动，并附带 0—1 个小特质。

小特质示例：

- 财眼：稀有物出现时物资欲临时提高。
- 犟骨：低生命时仍可能拒绝撤离。
- 记仇：被抢或被骗后仇恨下降更慢。
- 快腿：单走和撤离更快。
- 碎嘴：压力高时提高暴露。
- 护短：更愿意救另一名 AI，不一定救玩家。

## 15. AI 决策 MVP 公式

AI 不再计算“是否服从玩家指令”，而是计算多个个人意图的效用。

### 15.1 意图效用

```text
intent_utility =
基础值
+ 人格驱动力修正
+ 当前背包价值修正
+ 生命与压力修正
+ 位置修正
+ 玩家喊话修正
+ 信任/怀疑/仇恨修正
+ 事件标签修正
+ random(-10, 10)
```

### 15.2 单走搜索

```text
solo_search =
20
+ loot_drive × 0.35
+ reward_preview × 0.25
+ mobility × 0.10
- survival_drive × threat / 220
- current_weight × 0.10
+ chaos × 0.08
+ random(-10, 10)
```

## 16. 胜利、失败与结算

### 16.1 成功

玩家成功撤离即算本局成功，但收益取决于撤离姿态：

- 玩家背包带出多少。
- 是否带出任务物品。
- 是否带出 AI 队友。
- 是否抢了队友包。
- 是否丢包轻装撤。

### 16.2 失败

失败条件：

- 玩家死亡。
- 玩家倒地后无人救援且撤离窗口关闭。
- 玩家被困，无法到达撤离。
- 路线结束时玩家未撤离。

### 16.3 结算记录

MVP 至少记录：

- 带出物资价值。
- 丢弃物资价值。
- 抢包次数。
- 被 AI 抢包次数。
- 抛弃队友次数。
- 被队友抛弃次数。
- 玩家恶名。
- 关键事故时间线 3—5 条。

## 17. MVP 内容范围

### 必须实现

- 2 个核心场景：
  - 路线选择场景。
  - 事件/互坑场景。
- 5 层随机路线。
- 每层 2—4 个节点，包含分叉和汇合。
- 6 类节点：战斗、搜索、争执、事故、撤离、未知。
- 玩家是三人之一，不是队长。
- 2 名随机 AI 队友。
- 至少 6 种 AI 人格模板。
- 固定 10 个玩家行动。
- 至少 7 种互坑事故。
- 至少 12 个事件配置。
- 个人背包、负重、丢包、抢包。
- 轻量位置状态和队伍分裂。
- 多种撤离姿态。
- AI 意图与主要原因展示。
- 成功、失败和简易结算弹窗。

### 暂不实现

- 完整卡牌构筑。
- 完整指令库。
- 玩家作为队长的强指挥系统。
- 永久队友养成。
- 完整队友关系网。
- 装备词条。
- 复杂格子背包。
- 实时移动和射击。
- 联网合作或 PVP。
- 长剧情和语音。

## 18. MVP 实施顺序

1. 固定测试数据：默认玩家、两名 AI、10 个行动、基础背包。
2. 事件/互坑场景：先打通一个事件的意图展示、玩家行动、结算和结果反馈。
3. 实现个人背包、丢包和抢包。
4. 实现位置状态：同行、单走、倒地、撤离中。
5. 实现 AI 意图效用和原因展示。
6. 实现撤离姿态和简易结算。
7. 路线选择场景：先用固定路线选择下一节点。
8. 两个场景互相切换：路线 → 事件 → 路线。
9. 接入随机路线生成。
10. 扩展到 12 个事件配置和 6 种 AI 人格。
11. 接入 chaos、player_infamy 和互坑事故。

先把一个事件里的“AI 想坑、玩家也能坑、有人带包撤离”跑通，再扩展路线和内容。

## 19. 原型验收标准

- 启动后能直接进入路线选择场景。
- 路线图稳定生成 5 层，每层 2—4 个节点。
- 玩家能从可达节点中选择下一事件。
- 选择节点后进入事件/互坑场景。
- 玩家能看到两名 AI 的大致意图。
- 玩家能选择行动和一句喊话。
- AI 至少会出现跟随、单走、搜索、抢包、撤离、救援中的多种行为。
- 每次明显作恶能展示至少 1 条主要原因。
- 事件结算能更新生命、背包、位置、威胁、混乱、撤离窗口或恶名。
- 一局内能出现至少一次队伍分裂。
- 一局内能出现至少一次抢包、搜包、丢包或卖人。
- 玩家能通过丢包提高撤离成功率。
- 玩家能独自撤离并按个人背包结算收益。
- AI 能抢走物资或提前撤离。
- 战斗/事件结束后能返回路线选择场景。
- 玩家撤离后能显示结算。
- 玩家死亡或被困后能显示失败。
- 单局能在 5—10 分钟内完成。

这一版 MVP 的目标很窄：证明“三人野排互坑 + 背包归属 + 撤离姿态”这个核心是否好玩。只要这个骨架成立，后续再补主菜单、角色成长、卡牌系统和更复杂的事故链。
