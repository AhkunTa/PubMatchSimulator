# 《野排模拟器》游戏设计总览

> 文档版本：0.2  
> 类型：单人、回合制、事件驱动、黑色喜剧互坑撤离 Roguelite  
> 引擎：Godot 4.x

## 1. 产品愿景

《野排模拟器》希望还原一种熟悉、荒诞且略带恶意的野排体验：你和两个临时队友表面上是小队，实际上每个人都在盘算自己怎么活、怎么多拿、什么时候卖人。

玩家不是队长，也不是全局指挥官。玩家只是三人小队中的一员，只能对另外两名 AI 队友发出简单喊话、临时约定或欺骗性暗示。AI 队友不默认服从玩家，它们有自己的目标、贪念、恐惧和坏心眼；会单走、抢包、假救援、引怪、提前撤离、甩锅，也会在关键时刻突然立功。

游戏的乐趣不是“控制队伍”，而是在两个同样不可靠的队友之间坏得更聪明：什么时候合作，什么时候装好人，什么时候抢包，什么时候丢包跑路，什么时候把队友留给敌人。

## 2. 一句话定义

玩家扮演三人野排小队中的一员，在随机路线和事件中与两个高随机、高私心的 AI 队友互相利用、互相坑害，尽量带着物资活着撤离。

## 3. 目标体验

### 野排失控感

玩家不能稳定命令队友。队友可能不听、听反、假装听、半路单走，甚至因为自己的收益直接破坏玩家计划。

### 互坑喜剧

游戏鼓励玩家和 AI 做坏事。抢包、卖队友、丢包撤离、假装救援、引怪转移压力都不是失败边缘行为，而是核心玩法选择。

### 撤离赌局

活着出去不难，带着好东西出去才难。满载撤离、轻装撤离、抢包撤离、卖人撤离、丢包保命都应该是不同风味的结局。

### 可复述事故

一局结束后，玩家应该能讲出具体故事：

> “阿强单走舔空投引来一队，我和老周本来想救，结果老周先摸了他的包。我看撤离车快开了，直接丢了大包轻装跑，最后只带出一把破枪。”

## 4. 核心支柱

1. **玩家也是普通队友**：玩家没有稳定统治权，只能用喊话、行动、诱骗和临时利益影响局面。
2. **AI 主动作恶**：AI 不只是违抗命令，而是会主动抢、骗、跑、卖、救、怂、莽。
3. **互坑有收益也有后果**：坏事可以带来生存、物资或位置优势，但会制造仇恨、分裂、失去火力或引发反咬。
4. **背包归属决定收益**：物资绑定到角色身上。谁活着撤离，谁身上的东西才算带出。
5. **队伍分裂是常态**：三人同行只是局势之一，单走、二打一、两人结盟、倒地搜包、独自撤离都应频繁出现。
6. **随机性必须可解释**：AI 行为可以荒诞，但事前要给线索，事后要说明主要原因。

## 5. 完整游戏循环

### 局外

```text
选择玩家角色
→ 配置起始装备或默认背包
→ 匹配两名随机 AI 队友
→ 进入行动路线
→ 完成对局
→ 按撤离姿态结算带出物资、恶名、事故记录和角色熟练度
→ 解锁新的角色、起始装备、AI 人格、事件和挑战
```

### 局内

```text
认识两个随机队友
→ 选择路线节点
→ 进入事件
→ 三人各自产生意图
→ 玩家选择行动或简单喊话
→ 队伍合流、分裂、互坑或爆发事故
→ 更新生命、背包、压力、仇恨、位置和撤离窗口
→ 继续深入、尝试汇合、抢包、救人或撤离
```

## 6. 玩家决策层

每个事件节点包含四个阶段：

1. **情报阶段**：展示事件、敌情、收益、撤离状态、队友位置和队友倾向。部分信息可能模糊或错误。
2. **意图阶段**：两名 AI 根据人格、背包、生命、敌情和关系生成自己的行动意图。
3. **玩家行动阶段**：玩家选择自己的行动，也可以附带一句简单喊话或欺骗性承诺。
4. **结算阶段**：三人行动同时或按先后执行，产生合作、分裂、互坑、事故和状态变化。

玩家每回合不应完成大量微操作。单个节点目标操作时长为 20—45 秒。

## 7. 玩家可选行为

玩家的核心操作不是复杂指挥，而是选择自己怎么行动。

| 行为 | 作用 | 主要代价 |
| --- | --- | --- |
| 跟随 | 和某个队友保持同行 | 可能被带进坏局面 |
| 搜索 | 获取物资 | 暴露、贪婪和单走风险提高 |
| 进攻 | 抢节奏、压制敌人 | 受伤、噪音和队友误判风险 |
| 稳住 | 降低风险、观察队友 | 收益和节奏较低 |
| 救援 | 救倒地或受伤队友 | 可能被卖、被伏击或错过撤离 |
| 抢包 | 拿走队友或尸体物资 | 被发现后仇恨上升，可能内斗 |
| 丢包 | 放弃部分或全部物资轻装行动 | 带出收益下降 |
| 独自撤离 | 提前结束本局 | 带货有限，可能留下更高收益 |
| 引怪 | 把敌方压力转移到他人方向 | 可能反噬，恶名上升 |

玩家也可以发出简单喊话，例如：

- “跟我。”
- “撤了。”
- “别舔了。”
- “我去救。”
- “你顶一下。”
- “这包归你。”
- “我断后。”

喊话只影响 AI 判断，不保证服从。部分喊话可以是谎言，结算后由行为兑现或背刺。

## 8. AI 队友设计

AI 队友的目标是像两个随机路人玩家，而不是玩家的下属。

AI 由五层组成：

| 层 | 内容 | 是否跨局 |
| --- | --- | --- |
| 能力 | 战斗、搜索、救援、机动、携行 | 仅本局生成 |
| 私心 | 物资欲、生存欲、表现欲、报复欲、甩锅倾向 | 仅本局生成 |
| 人格模板 | 行为偏好和特殊反应 | 模板永久存在 |
| 当前局势 | 生命、压力、背包、位置、仇恨、恐慌 | 不跨局 |
| 关系印象 | 对玩家和另一名 AI 的信任、戒备、敌意 | 不跨局 |

AI 不应该只是“不听话”，而应该有多种主动行为：

- 单走搜索。
- 抢玩家或队友的包。
- 假装救援实际舔包。
- 高压下丢包逃跑。
- 撤离点快关闭时提前跑路。
- 把敌人引向别人。
- 和另一个 AI 临时结盟。
- 倒地时许诺分赃骗玩家救援。
- 看到玩家作恶后反咬。

AI 决策反馈分三级展示：

- 明确信息：“阿强盯着空投箱，明显想单走。”
- 概率信息：“老周可能会跟你撤，但他背包很满。”
- 事后解释：“老周提前撤离：低生命 + 背包高价值 + 不相信你会断后。”

## 9. 队伍分裂与位置

MVP 不需要复杂地图移动，但必须有轻量位置状态。

| 位置状态 | 含义 |
| --- | --- |
| 同行 | 与目标角色在同一交战/搜索区域 |
| 邻近 | 可以救援或支援，但有延迟和风险 |
| 分散 | 无法直接支援，容易形成少打多 |
| 单走 | 独自在另一区域行动，收益和风险都提高 |
| 倒地 | 可被救、被搜包或被抛弃 |
| 撤离中 | 正在脱离，需要时间或窗口 |
| 已撤离 | 本局对该角色结束 |

队伍分裂会直接改变战斗人数。例如队友单走后，玩家和另一名 AI 可能在主战区变成 2 打 3；也可能玩家单走搜到肥包，但撤离时没人掩护。

## 10. 背包与物资

物资不只是总数，而是绑定到具体角色。

```text
角色背包 =
  轻物资
  重物资
  高价值物
  队友包裹
  任务物品
```

背包影响：

- 撤离速度。
- 战斗和机动稳定度。
- 被队友盯上的概率。
- 丢包后的生存率。
- 死亡或倒地后的可搜刮价值。

核心规则：

- 只有成功撤离角色身上的物资才算收益。
- 倒地角色的包可以被救援者拿走，也可以被 AI 偷走。
- 丢包可以大幅提高撤离或逃跑成功率，但减少收益。
- 抢到高价值物会提高其他人的敌意和背刺概率。

## 11. 撤离姿态

撤离按姿态结算，不只是成功/失败。

| 撤离姿态 | 收益 | 风险 | 叙事结果 |
| --- | --- | --- | --- |
| 全队满载撤 | 最高 | 最高 | 罕见的真合作 |
| 两人带货撤 | 高 | 中高 | 抛下一个麻烦队友 |
| 独自带货撤 | 中高 | 中 | 玩家成功卖队 |
| 丢包轻装撤 | 低 | 低 | 活命优先 |
| 抢包撤 | 高 | 高 | 高收益高仇恨 |
| 被队友卖 | 低或失败 | 高 | 形成事故记录 |
| 全灭或被困 | 无 | 最高 | 高质量失败故事 |

撤离窗口可以是时间、条件或风险槽：

- 撤离车还需 2 回合到达。
- 直升机噪音会吸引敌人。
- 下水道只能轻装通过。
- 撤离点只够一个人先走。
- 背包过重会拖慢全队。

## 12. 角色系统

玩家角色不代表队长职业，而代表“你这个野排玩家擅长用什么方式活下来”。

| 角色方向 | 强项 | 作恶玩法 |
| --- | --- | --- |
| 突击位 | 正面突破、压制、抢节奏 | 用火力逼队友跟进，失败时甩锅 |
| 支援位 | 救援、治疗、装好人 | 以救援为名接近肥包 |
| 侦察位 | 识别风险、找撤离路 | 提前知道坏消息并单独跑路 |
| 工程位 | 背包、负重、丢包和撤离 | 管物资分配，也最方便偷换 |
| 流氓位 | 抢包、欺骗、引怪 | 明确偏向互坑和黑色喜剧 |

角色详细数值见[角色数值与成长](../systems/CHARACTER_AND_PROGRESSION.md)。

## 13. 事件设计模板

每个事件至少包含：

```text
事件 ID
事件类型
公开信息
隐藏信息
参与者位置
物资归属
AI 候选意图
玩家可选行动
喊话影响
互坑机会
撤离变化
结算规则
状态变化
反馈文本
后续事件标签
```

示例：

```text
事件：空投箱落在街口
公开信息：高价值物资、敌人可能靠近、撤离点还有 3 回合关闭
隐藏信息：箱子附近有伏击
AI 倾向：
  高物资欲：想单走舔箱
  高生存欲：想绕开
  高表现欲：想先打敌人
玩家机会：
  跟队友一起搜
  假装掩护实际抢包
  让队友先冲
  丢包轻装抢撤离
可能结果：
  合作拿货
  队友倒地
  玩家抢包
  AI 抢包跑路
  敌人包围导致 2 打 3
```

## 14. 难度设计

难度不只提高敌人生命，主要调整：

- AI 作恶频率和隐蔽程度。
- 队友单走倾向。
- 物资诱惑强度。
- 撤离窗口苛刻程度。
- 背包负重惩罚。
- 敌方追击和包围速度。
- 玩家作恶后的反噬强度。
- 信息模糊和谎言概率。

基础难度要允许玩家看懂“谁因为什么坏了事”。高难度才要求玩家主动利用 AI 私心、撤离姿态和背包归属。

## 15. 反馈与 UI 原则

- 决策前告诉玩家“队友大概想干什么”。
- 决策前告诉玩家“这件坏事可能带来什么收益”。
- 决策后告诉玩家“谁坑了谁，以及为什么”。
- 物资必须显示归属，不只显示总物资。
- 队伍位置必须清楚显示，避免分裂后玩家不知道谁在哪。
- 隐藏精确概率可以保留戏剧性，但不能隐藏关键影响因素。
- AI 台词服务于行为线索和黑色喜剧，而不只是装饰。

建议主界面布局：

```text
顶部：节点、威胁、撤离窗口、玩家背包价值、总队伍风险
左侧：两名 AI 的生命/压力/背包/位置/当前意图
中央：事件描述、位置状态和结果演出
底部：玩家行动与简单喊话
右侧：玩家状态、背包、撤离选项和当前恶名/仇恨
```

## 16. 版本路线

### 原型版

验证三件事：

1. AI 主动作恶是否有趣。
2. 玩家互坑选择是否形成明确收益和代价。
3. 背包归属与撤离姿态是否支撑“再贪一下/直接跑路”的循环。

范围以 [MVP](MVP.md) 为准。

### Alpha

加入更多角色、更多 AI 人格、队友之间的轻量结盟/仇恨、更多互坑事件链和基础局外解锁。

### Beta

加入完整局外成长、难度层级、事故图鉴、教程、结算时间线和表现优化。

### 正式版候选方向

每日行动、可分享对局种子、臭名挑战、极恶路线、低恶名合作路线。联网合作或 PVP 不作为默认路线，除非单人互坑核心已经稳定。

## 17. 待验证风险

- AI 作恶是否有趣，还是让玩家觉得被系统恶意惩罚。
- 玩家作恶是否有足够收益，而不是纯负面道德按钮。
- 队伍分裂是否可读，玩家是否清楚谁在哪、谁带着什么。
- 丢包撤离和独自撤离是否能形成有效取舍。
- AI 行为解释是否既清楚又不拖慢节奏。
- 互坑喜剧是否会压过 Roguelite 风险收益判断。
