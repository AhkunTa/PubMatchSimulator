## 统一术语

| 术语 | 定义 |
| --- | --- |
| 行动 | 玩家本回合真正执行的主要选择，例如搜索、救援、抢包、丢包、撤离 |
| 喊话 | 玩家对 AI 发出的简单口头影响，例如“跟我”“撤了”“包归你”，不保证生效 |
| 意图 | AI 当前最想执行的行为，例如单走搜索、提前撤离、抢包、救援 |
| 作恶 | 玩家或 AI 为了收益或自保主动坑害他人的行为 |
| 互坑事故 | 单走、抢包、假救援、引怪卖人、提前撤离等局势爆点 |
| 背包归属 | 物资绑定到具体角色身上，谁撤离谁带出 |
| 丢包 | 放弃部分或全部物资以提高逃跑、机动或撤离稳定度 |
| 抢包 | 从倒地、死亡、撤离中或同行队友身上夺取物资 |
| 恶名 | 玩家本局作恶后被 AI 防备、报复或甩锅的程度 |
| 局内信任 | 单局内 AI 对玩家当前判断和行为的信任 |
| 局外好感 | 特殊 ID 对玩家的长期关系评价 |
| 怨账 | 特殊 ID 对玩家长期食言、抢包、卖人等行为的记忆 |
| 默契 | 招募后固定队友与玩家长期组队形成的配合程度 |
| 特殊 ID | 可长期记录、触发专属事件并被招募的固定设定 AI |
| 固定队友 | 已招募的特殊 ID，可参与双排或三排 |
| 战斗卡牌 | 节点回合中主动打出的行动卡 |
| 事件卡牌 | 事故、招募、好感、撤离分歧等事件中出现的选择卡 |
| 威胁 | 当前对局整体危险水平 |
| 混乱 | 局势失控程度，越高越容易触发极端 AI 行为 |
| 撤离窗口 | 当前撤离机会的稳定程度，越低越难安全带货离开 |

## 维护规则

设计改动时，先修改负责该问题的主文档，再同步其他文档中的摘要。

- 好感、招募、双排、三排：优先改 [好感度与招募系统](systems/AFFINITY_AND_RECRUITMENT.md)。
- 战斗卡、事件卡、卡牌 JSON：优先改 [卡牌系统](systems/CARD_SYSTEM.md)。
- 玩家角色、AI 属性、特殊 ID 属性、成长：优先改 [角色数值与成长](systems/CHARACTER_AND_PROGRESSION.md)。
- 第一版可玩范围：优先改 [MVP 设计文档](core/MVP.md)。
- 代码模块和数据流：优先改 [技术架构](technical/SYSTEM_ARCHITECTURE.md)。
