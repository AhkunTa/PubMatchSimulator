# 《野排模拟器》设计文档索引

## 阅读顺序

1. [游戏设计总览](GAME_DESIGN.md)：产品定位、核心体验、完整循环和版本路线。
2. [MVP 设计文档](MVP.md)：首个可玩版本的严格范围与验收标准。
3. [角色数值与成长](CHARACTER_AND_PROGRESSION.md)：玩家职业、AI 人格、局内与局外成长。
4. [场景与界面流程](SCENE_DESIGN.md)：基础场景、随机路线、界面职责和场景流转。
5. [技术架构](SYSTEM_ARCHITECTURE.md)：Godot 模块划分、数据结构、流程和实现约束。

## 文档职责

| 文档 | 回答的问题 | 不负责 |
| --- | --- | --- |
| 游戏设计总览 | 游戏为什么有趣，长期要做成什么 | 具体代码结构 |
| MVP | 第一版做什么、如何判断成功 | 后期所有内容 |
| 角色数值与成长 | 数值如何作用、角色如何差异化 | UI 和存档实现 |
| 场景与界面流程 | 玩家在哪里做决定、场景如何衔接 | 最终美术风格 |
| 技术架构 | 系统如何拆分、数据如何流转 | 最终平衡数值 |

## 统一术语

| 术语 | 定义 |
| --- | --- |
| 指令 | 玩家表达战术意图的主要操作 |
| 服从 | AI 是否接受玩家当前指令 |
| 偏离 | AI 因个人动机选择其他行为 |
| 驱动力 | 物资欲、生存欲、表现欲等 AI 内在倾向 |
| 信任 | 单个 AI 对玩家的局内关系 |
| 凝聚力 | 全队作为整体保持协同的能力 |
| 压力 | 单个角色在危险下的失控程度 |
| 威胁 | 当前对局整体危险水平 |
| 局内成长 | 只在本局有效的状态和能力变化 |
| 局外成长 | 跨对局保留的解锁与熟练度 |

设计改动时，应先修改负责该问题的主文档，再同步其他文档中的摘要，避免多份规则互相冲突。
